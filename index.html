bm2000

发表于 2016-12-31 | 分类于 default |

闲来算算学桥牌已经有8年了。今天输了一场团体赛，仔细数数失误实在是不少。感觉还是不够认真，基本功还是不够，细节处理不好，和顶尖的差距大概就是这个了吧。年底开坑复习一下bm2000，其他不多说了。

3阶开始。12.31 mark第一副

6S，首攻HK。

做手里的牌，H2可以用CK垫掉。将牌都是顶张。DA，DK然后交叉将吃。乍一看是一个不错的7S的牌。

多年不打，第一次HA，CA，DK，CK，DA果然被将吃了。。回将牌宕1 - -

D51的情况下，方块回手的时候只能放小，然后交叉将吃两轮，将牌32即可mk。

略怪异，只是针对左手单张方块的打法，了解一下即可。

6NT，首攻DQ。

黑桃3个，红桃2个，方块2个。还需要5个赢墩需要从C树立。

这种结构需要担心的就是C的50。 如果手里先拔CA，右手5张的时候就会宕掉。

安全打法是从明手出C6，若东家没有，则西家5张。CA接住CT飞过即可。

若东家出大（CJ，C9），CA盖过，顶出一张大牌即可。 出小则也必须放小，任何情况下都可以mk。

guzzle增加重试

发表于 2016-12-30 | 分类于 default |

最近线上有时http请求会连接超时，需要增加重试逻辑。查看guzzle代码发现已经自带了一个RetryMiddleware，简单记录一下。

middleware需要一个decider确定重试逻辑，需要一个delay函数拉开重试间隔。
确定重试逻辑为连接错误、超时等和503， 重试最多两次。

debug的时候才发现php -S的时候这个server同时只能接收一个请求，大概就是1个进程的fpm… 稍微调了一下- -

网易云音乐小爬虫

发表于 2016-12-12 | 分类于 default |

平时一直使用网易云音乐听歌，一大原因就是喜欢看下面每个人的评论，就像看bilibili的时候一定要看弹幕一样。最近闲来无事，准备做一个简单的爬虫抓取一些热门的歌曲和评论，权当练手和娱乐，顺简单记录一下。

简单分析，页面内容由js渲染，简单调研了一下phantomjs和selenium之类的，最终使用phantomjs进行抓取。

网页内容较为规整。手工指定了抓取顺序。

  1. [http://music.163.com/#/discover/playlist/?order=hot&cat=全部&limit=35&offset=] 获取分类为全部按热门排序的歌单（playlist）
    共抓取1467个歌单。
  2. [http://music.163.com/#/playlist?id=] 获取歌单内的歌曲id. 共抓取110332首歌曲id，去重后为59919。本次抓取主要针对这些歌曲。
  3. [http://music.163.com/#/song?id=] 获取歌曲歌名，歌手，歌词，总评论数，热门评论，点赞数，最新评论，解析后入库。

js代码基本雷同，参考phantomjs官网即可。几个注意点是所有内容均在一个iframe里面，最初抓取的时候没有注意到，导致一度怀疑人生。。还是就是一定要设定一个超时时间，因为有时候不知道什么原因脚本就会hang住- -

通过一个shell脚本串联整个程序，没有多进程、重试等等。（因为我懒。。 其实是因为我的机器只有1核，多进程应该也没啥用。。）

每个页面大致需要1至2秒加载并解析，经过两天左右的时间… 结果如下：

再次闲到没事干- - 想自己捣鼓个框架出来。写了一点感觉对整体把握还是有些欠缺，先向业界经典看齐一下。

composer global require "laravel/installer"
laravel new blog

chmod -R o+w storage
chmod -R o+w bootstrap/cache

需要生成一个Application Key, 放在了.env里。

php artisan key:generate

数据库migration

php artisan make:migration create_tasks_table --create=tasks // database/migrations
php artisan migrate
php artisan make:model Task // app/Task.php

路由在app/Http/routes.php
模板在resources/views

生成登录页，还需要修改
app/Http/Controllers/Auth/AuthController
app/Http/Middleware/RedirectIfAuthenticated.php

php artisan make:auth

生成controller

php artisan make:controller TaskController
php artisan make:controller PhotoController --resource

生成policy

php artisan make:policy TaskPolicy
